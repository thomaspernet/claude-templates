# Style Guide Rules

## Tone

- Professional but approachable
- Authoritative without being condescending
- Prefer concrete examples over abstract explanations

## Sentences

- Average sentence length: 15-20 words
- Maximum: 25 words — split longer sentences
- One idea per sentence
- Active voice preferred ("The team built X" not "X was built by the team")

## Paragraphs

- One main point per paragraph
- 3-5 sentences typical
- Lead with the key point, then support it
- Transition between paragraphs

## Vocabulary

- Use plain language — avoid jargon unless writing for specialists
- Define technical terms on first use
- Be consistent — once you choose a term, use it everywhere
- Avoid: "utilize" (use "use"), "leverage" (use "use"), "synergy", "paradigm"

## Formatting

- Bulleted lists for 3+ related items
- Numbered lists for sequential steps
- Bold for key terms on first introduction
- Code formatting for technical identifiers
- Tables for structured comparisons

## Headings

- H1: Document title (one per document)
- H2: Major sections
- H3: Subsections (use sparingly)
- Never skip levels (no H1 → H3)
- Headings should be scannable — reader gets the gist from headings alone
