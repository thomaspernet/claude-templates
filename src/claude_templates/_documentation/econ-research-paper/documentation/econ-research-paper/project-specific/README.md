# Project-Specific Economics Research Paper Documentation

This folder tracks the current state of **this project's** economics research paper. It serves as both a human-readable reference and a benchmark the coding agent uses when drafting, reviewing, or improving sections.

## How to Use This Folder

1. **Fill in the files below** as your research progresses — research question, data, identification strategy, paper outline.
2. **Reference the general best practices** in `../general-best-practices/` as the benchmark for structure and writing quality.
3. **Keep docs in sync with the paper** — update this folder whenever you finalize a section, change your strategy, or receive reviewer feedback.

## Suggested Structure

| File | Purpose | When to Create |
|------|---------|----------------|
| `01-research-design.md` | Research question, WHAT/WHY/HOW answers, contribution | Always — start here |
| `02-data-and-variables.md` | Data sources, variable definitions, sample construction | When data is finalized |
| `03-empirical-strategy.md` | Identification strategy, threats, robustness plan | When strategy is chosen |
| `04-paper-outline.md` | Section-by-section outline with key arguments | Before first full draft |
| `05-revision-log.md` | Track feedback, revisions, and decisions | After first draft |

## Writing Guidelines

- **Be specific.** Reference actual table numbers, variable names, and data file paths.
- **Answer the skeptic.** For every design choice, note why it's the right one and what alternatives were considered.
- **Date your decisions.** Note when identification strategy, data, or scope changed and why.
- **Track reviewer feedback.** Log comments from advisors, referees, and seminar participants with your response.
