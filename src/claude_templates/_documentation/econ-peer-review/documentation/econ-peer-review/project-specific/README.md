# Project-Specific Peer Review Documentation

This folder tracks peer review activity for **this project's** economics papers — both reviews you write and reviews you receive.

## How to Use This Folder

1. **Track reviews you write** — maintain structured notes on papers you referee
2. **Track reviews you receive** — log every referee comment with status and response
3. **Reference the general best practices** in `../general-best-practices/` as the benchmark for review quality
4. **Keep docs in sync** — update response trackers as you address each comment

## Suggested Structure

| File | Purpose | When to Create |
|------|---------|----------------|
| `01-review-tracker.md` | Log of reviews received with status and response | When you receive a referee report |
| `02-revision-plan.md` | Priority-ordered plan for addressing referee comments | Before starting a revision |
| `03-response-letter.md` | Draft response letter to referees | During revision |

## Keeping This Folder Updated

- When you receive a referee report, immediately create `01-review-tracker.md` with every comment
- Mark each issue: `[OPEN]`, `[IN PROGRESS]`, `[DONE]`
- Do not delete resolved items — keep the full history for co-authors and future rounds
- Address every comment in the response letter, even to respectfully disagree
- Priority order: identification → missing analyses → structure → presentation → style → typos → citations
