# Consulting Presentation — Best Practices

## General Principles

Consulting presentations are standalone documents that must communicate a complete argument without a presenter. Executives read decks on planes, forward them to teams, and make decisions from them. Every slide must function independently while contributing to the overall narrative.

## The Pyramid Principle

Structure every deck top-down using Barbara Minto's Pyramid Principle:

1. **Lead with the answer.** State your recommendation on slide 1-2, not slide 30.
2. **Group supporting arguments.** 3-5 key reasons, each backed by evidence.
3. **MECE structure.** Arguments are Mutually Exclusive (no overlap) and Collectively Exhaustive (no gaps).

Never build up to a conclusion. Busy executives want the answer first, then the evidence.

## SCR Framework for Openings

Structure your executive summary using Situation-Complication-Resolution:

- **Situation**: The established context everyone agrees on
- **Complication**: What changed or what's at risk
- **Resolution**: Your recommendation (this is 60-70% of the exec summary)

## Slide Anatomy — Consulting Standard

Every content slide has exactly three parts:

| Element | Rule |
|---------|------|
| **Action title** | A complete sentence stating the slide's conclusion. Must pass the "titles only" test: reading all titles in sequence tells the full story. |
| **Body (exhibit)** | One chart, table, framework, or structured text that proves the action title. Never multiple unrelated exhibits. |
| **Source line** | Data source, date, methodology — in small font at bottom. Non-negotiable for credibility. |

## Deck Structure

| Section | Slides | Purpose |
|---------|--------|---------|
| Cover | 1 | Client name, project title, date, confidentiality |
| Executive summary | 1-2 | Full argument in SCR format — a reader who stops here gets 80% of the value |
| Situation / Context | 2-3 | Market data, current state, baseline metrics |
| Analysis / Findings | 5-10 | One insight per slide, evidence-based, MECE grouped |
| Recommendations | 2-3 | Specific, actionable, prioritized (impact × feasibility) |
| Implementation roadmap | 1-2 | Timeline, owners, milestones, quick wins |
| Appendix | 5-10 | Detailed analysis, methodology, data tables |

## Formatting Standards

- **Alignment**: All elements snap to grid. Misalignment signals sloppiness.
- **Boxes and borders**: Use sparingly. Grey hairline borders, not thick colored boxes.
- **Icons**: Flat, monochrome icons to label concepts. No clip art.
- **Frameworks**: 2×2 matrices, waterfall charts, Harvey balls for status — use standard consulting visuals.
- **Footnotes**: Numbered, bottom-left. Keep to 1-2 per slide.
- **Page numbers**: Bottom-right on every slide.

## Chart Types for Consulting

| Use Case | Chart Type |
|----------|-----------|
| Comparison across categories | Horizontal bar chart |
| Change over time | Line chart or vertical bar chart |
| Part-to-whole composition | Stacked bar or waterfall chart |
| Correlation between variables | Scatter plot |
| Process or decision flow | Flow diagram with boxes and arrows |
| Prioritization | 2×2 matrix (impact vs. effort) |
| Never | Pie charts (use bar charts instead), 3D anything |

## Common Mistakes to Avoid

- Topic titles ("Market Analysis") instead of action titles ("Market grew 12% but share declined")
- More than one message per slide
- Charts with no insight callout — every chart needs annotation highlighting the key finding
- "Boiling the ocean" — showing all analysis instead of curating the 5-6 most important findings
- Inconsistent formatting between slides (different fonts, colors, margin widths)
- No executive summary — executives may not read past slide 2

## Self-Review — Consulting Specific

After building the deck, verify:

- [ ] Executive summary works as a standalone document
- [ ] Titles-only test passes (read just titles = full argument)
- [ ] Every insight is MECE-grouped
- [ ] Recommendations are specific and actionable (who does what by when)
- [ ] Implementation has a timeline
- [ ] All charts have callout annotations highlighting the key insight
- [ ] Formatting is pixel-perfect: alignment, spacing, consistent styles
- [ ] Appendix has backup for every claim in the main body
- [ ] Page numbers on every slide
- [ ] Deck reads well without a presenter (standalone document test)
